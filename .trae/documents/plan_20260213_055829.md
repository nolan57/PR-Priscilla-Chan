## 修复 SpeechBrain 模型本地加载问题

### 问题
`hyperparams.yaml` 中的 `pretrained_path` 指向 HuggingFace 远程路径，导致 SpeechBrain 尝试从网络加载而非本地目录。

### 解决方案
修改 `_load_selected_model` 方法中 SpeechBrain 模型加载逻辑（约第 1805-1846 行）：

1. **动态修改 hyperparams.yaml**：在加载前读取 YAML 文件，将 `pretrained_path` 替换为本地目录路径
2. **使用修改后的配置加载**：使用 `Hparam` 类直接加载修改后的配置，而非通过 `from_hparams()`
3. **手动初始化模型**：根据配置手动构建模型并加载本地权重文件

### 修改位置
- 文件：`/Users/lpcw/Documents/PCS/audio-edit/pro_singer_separator.py`
- 方法：`_load_selected_model` 中的 `elif model_name == 'SpeechBrain':` 分支

### 关键改动
```python
# 读取并修改 hyperparams.yaml
with open(model_path / 'hyperparams.yaml', 'r') as f:
    hparams = yaml.safe_load(f)
# 将 pretrained_path 替换为本地路径
hparams['pretrained_path'] = str(model_path)
# 使用修改后的配置加载模型
```

这样可以确保 SpeechBrain 从本地目录加载所有模型文件，而不依赖 HuggingFace 缓存。
