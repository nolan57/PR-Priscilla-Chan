## Modify validate_mel_range to Validate Mel Without Clamp

### Problem Analysis
The current `validate_mel_range` function in `diagnose_mel.py` uses hardcoded clamp ranges (`MEL_MIN_DB` and `MEL_MAX_DB`) that are specific to clamped mel spectrograms. This validation is too restrictive for unclamped mel spectrograms that naturally have a wider dynamic range.

### Solution
Modify the `validate_mel_range` function to:
1. Remove clamp-specific validation logic
2. Simplify validation to check only that mel values are ≤ 0 (log-mel property)
3. Add a warning for extremely low values (potential numerical issues)
4. Remove unused `MEL_MIN_DB` and `MEL_MAX_DB` constants

### Implementation Steps
1. Edit `validate_mel_range` function to:
   - Change description to "验证 Mel 范围（应为 log-mel）"
   - Remove clamp-specific error messages
   - Check only that `mel_max ≤ 0.0`
   - Add warning for `mel_min < -30.0`
   - Update success message to "✅ Mel 范围正常（log-mel）"

2. Remove unused constants:
   - Delete `MEL_MIN_DB = -11.0`
   - Delete `MEL_MAX_DB = 0.0`
   - Update comment to "# NSF-HiFiGAN 期望的配置（来自 DiffSinger 官方预处理）"

This change makes the validation more flexible for unclamped mel spectrograms while still ensuring they're valid log-mel values.
